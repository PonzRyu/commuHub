# CommuHub ポートフォリオ

## 1. プロジェクト概要

### プロダクト名

- CommuHub（コミュハブ）

### 一言でいうと

- チームメンバーの予定を「週単位で横断的に把握」するための社内向けスケジュール共有 Web アプリです。

### 解決したい課題

- 個々のカレンダーを毎回開いて確認する手間を減らす
- 部署横断の会議調整や、週次運営の共有をスムーズにする
- 「誰が・いつ・何の予定があるか」を一覧で直感的に把握できる状態を作る

---

## 2. 担当範囲（実装）

- 要件定義に沿った UI / 機能実装
- 週間スケジュール一覧画面の設計・実装
- 管理画面（部署・メンバー・ICS リンク管理）の実装
- 週次共有事項 / 週次予定表編集機能の実装
- バリデーション・エラーハンドリング・セッション管理の実装
- データモデル設計（Prisma）と Next.js Server Action 連携

---

## 3. 主要機能

1. **ウィークリースケジュール表示**

- メンバーごとの週間予定を 1 画面で一覧表示
- 部署フィルタ、週移動、手動リフレッシュに対応

1. **ICS カレンダー連携**

- メンバー単位で ICS リンクを登録
- ICS を展開して当週の予定へ反映
- 予定なしメンバーの非表示切り替え

1. **並び替え UX**

- 部署順ソートとカスタム順を切り替え
- DnD（ドラッグ & ドロップ）で直感的に表示順を調整

1. **管理画面**

- 部署 CRUD
- メンバー CRUD（部署紐づけ、表示順、ICS リンク管理）
- 外部リンク管理、アプリ表示名変更

1. **週次運用機能**

- 週ごとの共有事項（テキスト）を保存
- 週ごとの予定表を JSON で保持し、編集可能
- メンバー予定から週次共有情報への取り込み機能

---

## 4. フロントエンド実装のアピールポイント

### 4-1. Next.js App Router の適材適所設計

- サーバー側で DB 取得・集約を行い、画面描画に必要な形へ整形
- インタラクションが必要な部分のみ `use client` を採用し、責務を分離
- Server Component / Client Component を混在させても保守しやすい構成を意識

### 4-2. 複雑 UI の安定動作

- DnD ライブラリ利用時の SSR/CSR 差分によるハイドレーション不一致を考慮
- マウント後のみ DnD を有効化する構成で、表示崩れや警告を回避

### 4-3. 入力体験とバリデーション

- ダイアログ編集 UI で、一覧画面から文脈を失わずに操作可能
- 必須入力・文字数・形式チェックを実装し、ユーザーに即時フィードバック
- エラー文言を UI に反映して再入力しやすい導線を整備

### 4-4. 実運用を想定したセキュリティ配慮

- 管理画面はセッションチェックを通過したユーザーのみアクセス可能
- リダイレクト先パスを安全化し、意図しない遷移を防止
- ICS URL はプロトコルやローカル/プライベート IP を検証し、危険な入力を抑制

---

## 5. 技術スタック

- **Frontend**: Next.js (App Router), React, TypeScript
- **UI**: Tailwind CSS, shadcn/ui, Radix UI, dnd-kit
- **Backend (BFF)**: Next.js Server Actions
- **DB / ORM**: PostgreSQL, Prisma
- **Calendar**: node-ical（ICS 展開）
- **Desktop 対応**: Electron（Windows 配布想定）

---

## 6. 画面構成（提出時にスクリーンショット添付推奨）

1. メイン: ウィークリースケジュール一覧
2. 管理: ログイン画面
3. 管理: 部署管理
4. 管理: メンバー管理（ICS 登録モーダル含む）
5. 週次運用: 共有事項 + 予定表編集

> 提出時は各画面のスクリーンショットを 1 枚ずつ添付すると、実装力と UI 品質が伝わりやすくなります。

---

## 7. 設計上の工夫

- 時刻・週計算を Tokyo 基準で統一し、週跨ぎの解釈差異を抑制
- 予定データは「表示用に整形した単位」で扱い、UI 側の複雑性を低減
- データ削除（空入力時の削除）と保存（upsert）を分けて、運用時の挙動を明確化
- 将来の拡張（部署追加、機能追加、デスクトップ配布）を見据えた構成

## 9. 応募時に添える説明文（そのまま利用可）

CommuHub は、チーム内の予定共有を効率化するために開発した Web アプリです。  
Next.js App Router を用いて、サーバー側のデータ集約とクライアント側の操作性を両立しました。  
特に、週間予定の一覧性、ICS 連携、管理 UI（CRUD + バリデーション）、DnD による並び替えなど、実運用を意識したフロントエンド実装に注力しています。  
また、管理セッションや URL 検証など、セキュリティ面の基本対策も組み込んでいます。
